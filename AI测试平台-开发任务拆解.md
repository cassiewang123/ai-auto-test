# AI 接口自动化测试平台 — 开发任务拆解

> 基于《AI 接口自动化测试平台开发计划》拆解。整体周期 **20 周（5 个月）**，分 5 个阶段、8 个核心模块、5 个里程碑。任务编号规则：`T{阶段}.{序号}`，子任务编号 `T{阶段}.{序号}.{子序号}`。

## 总览

| 维度 | 说明 |
|------|------|
| 技术栈 | FastAPI + PyTest + Playwright + Locust + React/TS + LangChain |
| 团队配置 | 后端 1-2 / 测试开发 1-2 / 前端 1 / AI 兼职 1 / DevOps 兼职 1 |
| 总工时基准 | 约 20 周（按 5 人并行估算，单人串行约 60+ 人周） |
| 里程碑 | M1 基础闭环 / M2 接口测试可用 / M3 压测+DB+监控 / M4 AI 上线 / M5 生产交付 |

任务依赖关系图（阶段间串行，阶段内可并行）：

```
Phase1 基础框架 ──▶ Phase2 核心功能 ──▶ Phase3 高级功能 ──▶ Phase4 AI增强 ──▶ Phase5 部署交付
   (4周)              (6周)              (4周)              (4周)            (2周)
```

## 阶段一：基础框架搭建（4 周，7/7 - 8/1）

**阶段目标**：搭建项目骨架，跑通"前端 → 后端 → 测试执行"最小闭环。
**阶段验收**：能在前端创建一条简单的 API 测试用例，点击执行后看到测试结果。
**对应里程碑**：M1 基础闭环（第 4 周末）。

## 关键风险与应对

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| Playwright API 能力与 UI 框架冲突 | 中 | 中 | Phase 1 即验证混合场景；封装统一 fixture；第 4 周评审决定是否降级 httpx 方案 |
| Locust 分布式压测资源不足 | 中 | 中 | K8s 弹性扩缩容；低并发单机即可；高并发按需启动 worker Pod |
| LLM 生成用例质量不稳定 | 中 | 中 | 生成代码必经 ruff 检查 + 人工审核；提供生成-预览-编辑-保存流程；积累 Prompt 模板 |
| 数据库测试数据污染 | 低 | 中 | 默认事务回滚；testcontainers 一次性容器；生产环境只读连接 |
| SSH 连接安全风险 | 低 | 中 | 密钥存 Vault/K8s Secret；连接白名单 IP；操作审计日志 |
| 开发周期延期（AI 模块复杂度超预期） | 中 | 中 | AI 模块可解耦独立迭代；先规则引擎再接 LLM；MVP 先支持用例生成 |
| 前端实时性不足（压测指标推送延迟） | 低 | 中 | WebSocket 推送 + InfluxDB 聚合；降级定时轮询；采样周期可配置 1-10s |

## 编码规范摘要

| 类别 | 工具 | 规则 |
|------|------|------|
| Python 格式化 | ruff + black | 行宽 100，自动排序 import |
| Python 类型检查 | mypy | 所有函数必须有类型注解 |
| Python 文档 | docstring（Google 风格） | 公开函数/类必须有文档字符串 |
| 前端格式化 | Prettier + ESLint | 2 空格缩进，单引号，尾逗号 |
| Git 提交 | Conventional Commits | `feat:`/`fix:`/`test:`/`docs:` 前缀 |
| 测试命名 | `test_{动作}_{预期结果}` | 如 `test_create_user_returns_201` |
| Fixture 命名 | snake_case | 如 `api_request_context`、`db_session` |
